# Code Of Conduct

Zynum contributors are expected to keep technical discussion respectful,
focused, and evidence-based.

Unacceptable behavior includes harassment, personal attacks, repeated
off-topic disruption, publishing private information, or intentionally
misrepresenting benchmark or correctness results.

Maintainers may moderate issues, discussions, pull requests, or other project
spaces to protect the quality and safety of the community. Enforcement actions
may include warnings, temporary restrictions, or removal from project spaces.

Report conduct concerns privately to the project maintainer, Kaixiang Huang, via
project-maintainer channels on <https://github.com/kaix-huang/Zynum>. Security
vulnerabilities should follow `SECURITY.md` and should not be disclosed in public
issues or discussions.
